Autómatas, Máquinas de Turing y su estructura algebraica

Dos secciones, un mismo análisis: para cada autómata (AFD mínimo compilado desde una expresión regular) y para cada Máquina de Turing (modelo estándar §6.1 de De Castro), la página calcula las propiedades de álgebra abstracta que le corresponden — el monoide de transición M(A), sus idempotentes, unidades y centro, si es grupo (ℤ/nℤ, V₄, S₃, …), si es abeliano o cíclico, y si es aperiódico (definido con las potencias de sus elementos). Precedencia (§2.2, de mayor a menor): * > concatenacion > union | (≡ ∪). Usa parentesis para forzar agrupamiento: (0|1)* ≠ 0|1*.

Alfabeto Σ: si lo dejas vacio, se infiere de los literales que aparecen en la expresion. Para usar el comodin . con simbolos que no aparecen literalmente, especifica Σ a mano (por ejemplo: 01 o a,b,c).

Importante: las expresiones regulares solo describen lenguajes regulares (Cap 2 del libro). Lenguajes como anbn no son regulares (§3.1); para esos, baja a la sección de Máquinas de Turing.

Cómo combinar restricciones. Sepáralas con coma. Se admite:
  • , n >= 1 (excluye n = 0, es decir, la palabra vacía λ). Ejemplo:
  • , m >= n (relaciones >=, <=, = entre dos variables).
  • Ambas a la vez:
  • Sinónimos válidos: where, donde, ;. Los signos > y < se convierten internamente a las versiones estrictas equivalentes (m > n ≡ m >= n con n >= 1).
La restricción n >= 1 no aplica a regex: ahí la cota la fija la propia expresión (p. ej., (0|1)+ en vez de (0|1)*).
