Cómo combinar restricciones. Sepáralas con coma. Se admite:
  • , n >= 1 (excluye n = 0, es decir, la palabra vacía λ). Ejemplo:
  • , m >= n (relaciones >=, <=, = entre dos variables).
  • Ambas a la vez:
  • Sinónimos válidos: where, donde, ;. Los signos > y < se convierten internamente a las versiones estrictas equivalentes (m > n ≡ m >= n con n >= 1).
La restricción n >= 1 no aplica a regex: ahí la cota la fija la propia expresión (p. ej., (0|1)+ en vez de (0|1)*).

Por defecto se admite n = 0 (palabra vacía incluida). Las construcciones siguen los algoritmos del §6.2 del libro y generalizaciones del mismo estilo (marcar–emparejar–rebobinar); los lenguajes regulares pasan por el Teorema 6.1.1 (AFD → MT).
